## III. ARCHITECTURE HEALTH

### Strengths ✅
- **All 43 pages lazy-loaded** — good initial load time
- **15 domain-specific contexts** — recently split from monolithic (this session)
- **Heavy libs dynamic-imported** — jsPDF, kuroshiro, pptxgenjs code-split
- **Firebase real-time sync** — Firestore for live multiplayer games
- **PWA support** — Service worker, manifest, offline indicator
- **Error tracking** — Sentry with 10% sampling
- **TypeScript strict** — 30+ type definition files

### Concerns ⚠️
- **No React Router** — custom URL-based routing via `useUrlRouter()`. Works but fragile for deep linking, back button, browser history
- **AppContent prop drilling** — destructures 80+ properties from contexts, passes to pages
- **273K LOC** — large for single-dev maintenance. No modular package boundaries
- **45 `console.log`** — debug statements in production code
- **489 commented-out code blocks** — significant cleanup needed
- **162 `eslint-disable`** — indicates lint rules too strict or code not conforming
- **33 files > 500 lines** — refactor candidates (listening-audio-view 918 LOC, App.tsx 992 LOC)

### Missing ❌
- **No React Router** — no browser history, deep links broken
- **No i18n** — Vietnamese-only, all strings hardcoded
- **No engagement analytics** — only Sentry error tracking, no Mixpanel/GA
- **No component documentation** — no Storybook
- **No CI/CD pipeline visible** — no GitHub Actions config found
- **No database schema documentation**

## IV. UX ASSESSMENT

### Navigation
- **Sidebar**: 7 sections, 30+ items — **overloaded for mobile**
- **Click depth**: Home → any feature = 1-2 clicks ✅
- **Game access**: Home → Game Hub → Select = 2 clicks ✅
- **No breadcrumbs** in nested views (Level → Lesson → Card)
- **No favorites/quick access** customization

### Onboarding — ✅ WELL DONE
- 11-step visual tour
- JLPT level selection on first login
- Touch + keyboard navigation
- Can re-access anytime
- Daily missions introduce engagement loop

### Design Consistency — ✅ STRONG
- Dark glassmorphism theme across all study pages
- Consistent color palette: #8b5cf6 (purple), #ec4899 (pink), dark bg
- Glass elements: `backdrop-filter: blur(12px)`
- Gradient dividers in settings modals
- 205 media query breakpoints for responsiveness

### Pain Points
1. **Sidebar navigation crowded** — needs collapsible sections or role-based filtering
2. **VIP upgrade path unclear** — no direct payment, "contact admin" friction
3. **Loading states inconsistent** — some pages show spinner, others don't
4. **Empty states sparse** — only 3 dedicated empty-state components
5. **No search** — can't search across vocabulary, lessons, or content
6. **No keyboard shortcuts** — power users can't navigate efficiently
